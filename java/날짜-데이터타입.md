# java에서 날짜 데이터 다루기
java에서 날짜 데이터를 다룰 경우 `java.util.Date / Calendar`에서 `java.time` 패키지로 발전해왔다. java8부터는 `java.time`이 권장되는데 이전 `Date`타입과 달리 **불변 객체이고 thread-safe**하기 때문이다.

### 예시1
#### 기존 Date
```java
import java.util.Date;

public class Main {
    public static void main(String[] args) {
        Date date = new Date();
        date.setTime(date.getTime() + 1000L * 60 * 60 * 24); // 하루 추가
    }
}
```
`setTime()`처럼 기존 객체의 값을 직접 변경하기 때문에 안전하지 않다.

멀티스레드에서도 문제가 발생한다.
```java
import java.util.Date;

public class Main {
    static Date date = new Date();

    public static void main(String[] args) {
        Runnable task = () -> {
            date.setTime(date.getTime() + 1000);
            System.out.println(date);
        };

        new Thread(task).start();
        new Thread(task).start();
    }
}
```
여러 스레드가 같은 객체의 내부 값을 동시에 수정하기 때문에 실행 순서에 따라 값이 꼬일 수가 있다.

#### java.time.LocalDate (권장)
```java
import java.time.LocalDate;

public class Main {
    public static void main(String[] args) {
        LocalDate date1 = LocalDate.of(2026, 3, 3);
        LocalDate date2 = date1.plusMonths(1);

        System.out.println(date1); // 2026-03-03
        System.out.println(date2); // 2026-04-03
    }
}
```
plusMonths()를 호출해도 기존 객체 date1은 변경되지 않는다. 즉, 내부값이 수정되는 것이 아니라 새로운 객체를 생성해서 반환하여 어떤 코드가 date1을 공유하고 있어도 값이 바뀌는 일은 없다.

```java
import java.time.LocalDate;

public class Main {
    static LocalDate date = LocalDate.of(2026, 3, 3);

    public static void main(String[] args) {
        Runnable task = () -> {
            LocalDate newDate = date.plusDays(1);
            System.out.println(newDate);
        };

        new Thread(task).start(); // 2026-03-04
        new Thread(task).start(); // 2026-03-04
    }
}
```
두 스레드가 동시에 실행돼도 새 `LocalDate` 객체를 만들어 반환하기 때문에 값을 보장할 수 있다.

### formatter
formatter 역시 **thread-safe**를 위해 구버전인 `SimpleDateFormat`보다 `DateTimeFormatter`가 권장된다. `DateTimeFormatter`은 `java.time.format` 패키지에서 제공된다.
